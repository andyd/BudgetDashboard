# Work Logs

## Overview

This directory contains daily work logs tracking project progress, decisions, and issues.

## Log Format

Each log follows a structured format:

- **Session Overview**: Date, time, focus areas
- **Tasks Completed**: Detailed list of accomplished work
- **Current Project State**: Project status and structure
- **Issues Identified**: Problems and their priority levels
- **Next Session Goals**: Planned tasks for upcoming work
- **Decision Records**: Important project decisions and rationale
- **Time Tracking**: Effort estimation and session duration

## Work Tracking Guidelines

### Daily Logs

- Create new log file for each work session
- Use date format: `YYYY-MM-DD.md`
- Update project status regularly
- Document all major decisions

### Issue Tracking

- **High Priority**: Blocking issues, security concerns, critical functionality
- **Medium Priority**: Important features, performance improvements, documentation
- **Low Priority**: Nice-to-have features, minor improvements, future enhancements

### Decision Records

Document all significant decisions including:

- Technology choices
- Architecture decisions
- Design patterns
- Process changes

### Time Tracking

- Track time spent on major tasks
- Note efficiency improvements
- Identify areas needing more focus

## Usage

1. Start each session by reviewing the latest log
2. Create new log file for current date
3. Update project status and progress
4. Document new issues and decisions
5. Plan next session goals before ending
